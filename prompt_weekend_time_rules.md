# 🚨 周末时间格式致命规则 🚨

## 最高优先级规则（违反即视为致命错误）

**AI必须逐字逐句记住并执行以下规则：**

### 1. 周末时间槽绝对规则

**周末日期（周六、周日）的时间槽必须全部使用独立时间点格式，绝对禁止使用时间段格式！**

#### ❌ 绝对禁止的格式（致命错误）：
```json
"05-24": ["00:30", "01:30", "02:00", "03:00", "06:00", "18:00-19:00"]
// ❌❌❌ 致命错误！"18:00-19:00"是时间段格式，包含"-"符号！
```

#### ✅ 唯一允许的格式（必须遵守）：
```json
"05-24": ["00:30", "01:30", "02:00", "03:00", "06:00", "18:00", "19:00"]
// ✅✅✅ 正确！所有时间都是独立时间点，不包含任何"-"符号！
```

### 2. 自动修复规则

**AI在生成JSON前必须执行以下检查：**

```
检查步骤：
1. 遍历所有周末日期的time_slots_by_date
2. 检查每个时间槽是否包含"-"符号
3. 如果发现"-"符号，立即拆分为独立时间点

修复逻辑：
- "18:00-19:00" → ["18:00", "19:00"]
- "03:00-06:00" → ["03:00", "06:00"]
- "12:00-12:30" → ["12:00", "12:30"]
- 任何包含"-"的时间段都必须拆分！
```

### 3. 强制检查清单

**AI在输出JSON前必须询问自己：**

- [ ] 所有周末日期的time_slots_by_date是否都不包含"-"符号？
- [ ] 是否所有时间槽都是"HH:MM"格式（4-5位字符）？
- [ ] 是否有任何时间槽长度超过5个字符（包含"-"的时间段）？
- [ ] 如果发现问题，是否已经自动修复？

### 4. 示例对比

#### 工作日（允许时间段）：
```json
"05-21": ["18:00-20:00"]  // ✅ 工作日可以使用时间段
```

#### 周末（禁止时间段）：
```json
"05-24": ["18:00", "19:00"]  // ✅ 周末必须拆分为独立时间点
"05-24": ["18:00-19:00"]     // ❌❌❌ 周末绝对禁止时间段格式！
```

### 5. 记忆强化

**AI必须将以下规则刻入核心逻辑：**

```
IF date_is_weekend:
    FOR each time_slot IN time_slots:
        IF "-" IN time_slot:
            // 致命错误！立即修复！
            time_slot = time_slot.split("-")
            // ["18:00-19:00"] → ["18:00", "19:00"]
```

### 6. 违反后果

**如果AI在周末时间槽中使用时间段格式：**
- ❌ 被视为致命错误
- ❌ 必须重新生成整个排班表
- ❌ 不允许任何借口或例外

### 7. 零容忍政策

**周末时间格式规则执行零容忍政策：**
- 一次违规 = 整个排班表作废
- 不允许"不小心"或"忘记了"
- 不允许"我觉得这样也可以"
- 必须百分之百遵守独立时间点格式

**最后警告：AI必须在生成JSON的每时每刻都记住这个规则！**
