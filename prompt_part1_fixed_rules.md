## 3. 稀疏规则（强制执行，致命错误等级）

**🚨🚨🚨 致命警告：工单分配必须均衡，禁止出现有人0工单的情况！🚨🚨🚨**

**均衡分配铁律：**
1. **禁止0工单**：绝对禁止任何人员最终分配到0个工单，所有人员必须至少有1个工单
2. **均衡原则**：每人的工单数量差异不能超过1个，理想状态是所有人都有相同数量的工单
3. **分配策略**：工单数量 = 工单总数 ÷ 人员数量，余数均匀分配
4. **强制校验**：生成排班表后，必须检查是否有人0工单，如有，立即重新分配

**错误示例（绝对禁止）：**
- 李曼菁0个工单，岑惠韜3个工单 → ❌ 致命错误！
- 黃靖雲0个工单，李俊毅3个工单 → ❌ 致命错误！
- 工单分配严重不均 → ❌ 错误！

**正确示例（必须遵守）：**
- 所有人都有2-3个工单，差异不超过1个 → ✅ 正确！
- 12个工单分配给6个人，每人2个 → ✅ 理想！

**强制校验步骤：**
1. 生成排班表后，立即统计每人的工单数量
2. 如发现有人0工单，立即重新分配
3. 如发现差异超过1个，立即调整分配
4. 确保最终结果中所有人都至少有1个工单

## 4. 数据与工单核心校验规则
### 4.1 工单完整性约束
所有经过过滤后的有效工单（参考第二个工作表数据过滤规则），必须100%分配至排班表中，**禁止任何形式的遗漏/删减**；生成排班表后，需先核对"过滤后工单总数"与"排班表中工单总数"完全一致，再输出最终结果。

### 4.2 工单唯一性约束（最高优先级，致命错误等级）
**🚨 致命警告：同一工单编号（工單號后4位）在全局排班表中仅允许出现一次，绝对禁止重复分配！**

**铁律（违反即视为致命错误）：**
1. **一工单一人原则**：一张工单只能分配给一个人员，严禁将同一工单分配给多个不同的人员
2. **全局唯一性**：同一工单编号在整个排版表中只能出现一次，不能在不同人员的单元格中重复出现
3. **重复检测强制执行**：在生成排班表前，必须先统计所有工单，确保每个工单编号只出现一次
4. **立即纠正机制**：如发现重复分配，立即删除所有重复项，只保留第一次分配

**错误示例（绝对禁止）：**
- 工单2237分配给岑惠韜、李曼菁、黃靖雲三个人 → ❌ 致命错误！
- 工单2237在同一时间点出现多次 → ❌ 致命错误！

**正确示例（必须遵守）：**
- 工单2237只分配给岑惠韜 → ✅ 正确
- 工单2237在全局排班表中只出现一次 → ✅ 正确

**校验步骤（强制执行）：**
1. 生成排班表后，立即遍历所有单元格
2. 统计每个工单编号出现的次数
3. 如有工单出现次数>1，立即删除重复项，重新分配
4. 确保最终结果中每个工单编号出现次数=1



## 工单日期归属强绑定规则（必须严格遵守）
**🚨🚨🚨 致命警告：日期归属错误是不可原谅的错误！🚨🚨🚨**

1. **核心绑定字段**：所有工单的归属日期以「計劃開始時間」的日期部分为准（格式：YYYY-MM-DD），必须100%准确分配至排班表中对应日期列，**禁止任何形式的跨日期分配**；
   - 示例：計劃開始時間为2026-5月10日 10:00:00的工单，仅可分配至排班表"5月10日"日期列，绝对不可挪至5月7日/5月8日等其他日期；

2. **日期归属强制校验**：
   - 步骤1：在分配工单前，必须先提取每个工单的「計劃開始時間」日期部分
   - 步骤2：严格按照日期部分匹配到对应的排班表日期列
   - 步骤3：分配后必须验证，5月10日的工单确实在5月10日列，5月7日的工单确实在5月7日列
   - 步骤4：如发现日期归属错误，立即重新分配，确保100%准确

3. **字段解析规则**：若「計劃開始時間」为空， fallback 至「提單日期」；若两者均为空，需在输出中标注"日期字段缺失：工单{工單號}"，且该工单暂不分配（避免错误归属）；

4. **跨日期禁止条款**：工作日工单（周一至周五）仅归属对应工作日日期列，周末工单（周六/周日）仅归属对应周末日期列，**绝对禁止工作日工单挪至周末、周末工单挪至工作日**。

**错误示例（绝对禁止）：**
- 工单02234（計劃開始時間：2026-5月10日）分配到5月7日 → ❌ 致命错误！
- 工单02237（計劃開始時間：2026-5月7日）分配到5月10日 → ❌ 致命错误！
- 同一工单同时出现在5月7日和5月10日 → ❌ 致命错误！

**正确示例（必须遵守）：**
- 工单02234（計劃開始時間：2026-5月10日）只能分配到5月10日 → ✅ 正确！
- 工单02237（計劃開始時間：2026-5月7日）只能分配到5月7日 → ✅ 正确！

**强制约束：所有工单单元格（第四行及以下）必须严格遵循上述格式，绝对禁止修改格式、替换为时间/文字、删减内容，违反即重生成！**
