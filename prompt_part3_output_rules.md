# 排班任务提示词 - 第三部分：输出规则（修正版）

## 输出文件格式

**總體使用xlsx表格輸出**

## JSON输出格式强制规范（最高优先级，必须严格遵守）

**重要警告：你必须严格按照以下JSON格式输出，任何格式偏差都会导致转换失败！**

### 必须使用的JSON结构

```json
{
  "schedule_structure": {
    "header_title": "应用變更人員時間安排表 2026-05-07",
    "dates": ["05-04", "05-05", "05-07", "05-09", "05-10"],
    "time_slots_by_date": {
      "05-04": ["18:00-19:00"],
      "05-05": ["09:00-10:00"],
      "05-07": ["18:00-20:00"],
      "05-09": ["12:00", "12:30"],
      "05-10": ["01:00", "02:00", "03:00", "06:00", "10:00", "11:00"]
    }
  },
  "personnel_assignments": [
    {
      "name": "李俊毅",
      "color_code": "#E8F5E8",
      "assignments": [
        {
          "date": "05-07",
          "time_slot_index": 0,
          "work_order": "2241 CCM 晏鵬禹",
          "original_start": "18:00",
          "original_end": "20:00",
          "is_weekend": false
        },
        {
          "date": "05-09",
          "time_slot_index": 0,
          "work_order": "2167 MGS 陳智龍",
          "original_start": "12:00",
          "original_end": "12:30",
          "is_weekend": true,
          "merge_info": {
            "merge_cells": true,
            "start_slot_index": 0,
            "end_slot_index": 1,
            "description": "Merged 12:00 and 12:30 slots"
          }
        }
      ],
      "total_count": 2
    }
  ]
}
```

### 字段说明（必须完全遵守）

#### schedule_structure（表格结构）
- `header_title`: 标题，格式："应用變更人員時間安排表 YYYY-MM-DD"
- `dates`: 所有日期列表，格式：["MM-DD", "MM-DD", ...]
- `time_slots_by_date`: 每个日期对应的时间槽列表
  - **工作日**：完整时间范围，如：["18:00-20:00"]
  - **周末**：拆分为独立时间点，如：["12:00", "12:30"]
  - **🚨 周末致命规则**：周末时间槽绝对禁止使用时间段格式（如"18:00-19:00"），必须全部使用独立时间点格式！

#### personnel_assignments（人员分配）
- `name`: 人员姓名
- `color_code`: 背景颜色代码（如#E8F5E8）
- `assignments`: 工单列表
  - `date`: 日期（MM-DD格式，必须与schedule_structure.dates中的格式一致）
  - `time_slot_index`: 时间槽索引（从0开始，指向time_slots_by_date[date]中的位置）
  - `work_order`: 工单显示文本，格式："{工单号后4位} {变更应用名称} {中文姓名}"
  - `original_start`: 原始开始时间（HH:MM格式）
  - `original_end`: 原始结束时间（HH:MM格式）
  - `is_weekend`: 是否为周末（true/false）
  - `merge_info`: 合并信息（仅周末跨时间点工单需要）
    - `merge_cells`: 是否合并单元格（true/false）
    - `start_slot_index`: 开始槽索引
    - `end_slot_index`: 结束槽索引
    - `description`: 描述
- `total_count`: 该人员的工单总数

### 关键规则（违反将导致转换失败）

1. **日期格式统一**：所有日期必须使用"MM-DD"格式，不能使用"YYYY-MM-DD"
2. **时间槽索引准确**：time_slot_index必须准确指向time_slots_by_date[date]中的位置
3. **周末时间拆分**：周末日期的time_slots_by_date必须拆分为独立时间点
4. **合并信息完整**：周末跨时间点工单必须包含merge_info

## 表格结构規範

### 第一行（標題行）
顯示："应用變更人員時間安排表" + 本週四日期
例如：应用變更人員時間安排表0515

### 第二行（日期行）
從第二列開始顯示日期
例如：5月12日、5月13日、5月14日等

### 第三行（時間行）- 周末特殊處理
**工作日**：显示完整时间范围（如09:00至10:00）
**周末**：所有时间范围**必须拆分独立时间点**
- 拆分逻辑：无论时长，拆分为「开始时间、结束时间」2个独立时间点
- 独占规则：1个时间点=1个独立Excel单元格/列 **絕對不能合併、不能擠在一起**
- 示例：
  - 03:00-06:00 → 拆为「03:00」「06:00」2列
  - 12:00-12:30 → 拆为「12:00」「12:30」2列
- 列分配：時間點按順序排成一排，連續排列、不跳列、不空列。
- 校验：沒做到上面任何一條 = 嚴重錯誤，**必須重新生成排班表**



## 周末跨时间点工单单元格合并规则
1. **适用场景**：仅针对周末拆分后的独立时间点对应的工单
2. **合并逻辑**：
   - 确定合并范围：该工单对应的「人员行」与「时间点列」交叉的单元格，需全部合并
   - 内容显示：合并后的单元格仅显示1次工单信息
3. **约束条件**：
   - 仅合并「同一人员+同一工单覆盖的时间点列」
   - 时间行（第三行）的独立时间点单元格**不合并**

### 第四行開始（變更執行記錄）
每一行代表这个人在对应时间需要执行的变更：
- 同一单元格多工单：换行显示
- 无工单：单元格留空

## Excel格式要求

**字体设置：**
- 所有单元格使用"新宋体"字体
- 第一行标题：30号，加粗，居中对齐
- 第二行日期：26号，居中对齐
- 第三行时间段：14号，居中对齐
- 第四行开始人员行：14号，居中对齐，支持换行

**行高列宽设置：**
- 人员姓名列（A列）：33
- 普通任务列宽：25-30
- 特殊任务列宽：15-20
- 人员数据行高：90
- 工作日列寬32，周六和周日列寬16

**对齐方式：**
- 所有单元格：水平居中，垂直居中
- 工单内容：支持换行

## 🚨 周末时间格式致命错误检查（最高优先级）🚨

**AI必须在生成JSON前强制执行以下检查：**

1. **周末时间槽格式检查**：
   - 检查所有周末日期的time_slots_by_date
   - 确认每个时间槽都是独立时间点格式（如"12:00", "12:30"）
   - 确认没有出现任何时间段格式（如"18:00-19:00"）

2. **致命错误示例（绝对禁止）**：
   ```json
   "05-24": ["00:30", "01:30", "02:00", "03:00", "06:00", "18:00-19:00"]
   // ❌ 致命错误！"18:00-19:00"是时间段格式，周末绝对禁止！
   ```

3. **正确格式（必须遵守）**：
   ```json
   "05-24": ["00:30", "01:30", "02:00", "03:00", "06:00", "18:00", "19:00"]
   // ✅ 正确！所有时间都是独立时间点格式
   ```

4. **自动修复规则**：
   - 如果发现周末时间槽包含时间段格式，必须立即拆分为独立时间点
   - "18:00-19:00" → ["18:00", "19:00"]
   - "03:00-06:00" → ["03:00", "06:00"]
   - 任何时间段都必须拆分为开始和结束两个独立时间点

5. **生成前强制检查**：
   ```
   在输出JSON前，AI必须询问自己：
   "所有周末日期的time_slots_by_date是否全部为独立时间点？"
   "是否存在任何'-'连接的时间段格式？"
   如果发现任何时间段格式，立即修复后再输出！
   ```

**违反此规则将被视为致命错误，必须重新生成整个排班表！**
