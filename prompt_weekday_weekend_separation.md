# 🚨 工作日/周末工单分离致命规则 🚨

## 最高优先级规则（违反即视为致命错误）

**AI必须逐字逐句记住并执行以下规则：**

### 1. 工作日/周末绝对分离原则

**工作日工单（周一至周五 18:00-20:00）绝对禁止分配到周末日期列！**
**周末工单（周六、周日任意时间）绝对禁止分配到工作日日期列！**

### 2. 致命错误示例（绝对禁止）

#### ❌ 错误示例1：工作日工单跑到周末
```
工单：2367 GBS 葉俊濠
計劃開始時間：2026-05-21（周四，工作日）
错误分配：分配到了05-24（周六，周末）
❌❌❌ 致命错误！工作日工单绝对禁止分配到周末！
```

#### ❌ 错误示例2：周末工单跑到工作日
```
工单：2385 AAS 張雋宇
計劃開始時間：2026-05-24（周六，周末）
错误分配：分配到了05-21（周四，工作日）
❌❌❌ 致命错误！周末工单绝对禁止分配到工作日！
```

### 3. 正确分配示例（必须遵守）

#### ✅ 正确示例1：工作日工单
```
工单：2367 GBS 葉俊濠
計劃開始時間：2026-05-21（周四，工作日）
正确分配：必须分配到05-21（周四）列
✅✅✅ 正确！工作日工单分配到工作日
```

#### ✅ 正确示例2：周末工单
```
工单：2385 AAS 張雋宇
計劃開始時間：2026-05-24（周六，周末）
正确分配：必须分配到05-24（周六）列
✅✅✅ 正确！周末工单分配到周末
```

### 4. 强制检查逻辑

**AI在分配每个工单前必须执行以下检查：**

```python
# 伪代码：AI必须执行的检查逻辑
def check_work_order_assignment(work_order, target_date):
    # 获取工单的计划开始时间
    planned_start = work_order['計劃開始時間']
    planned_date = extract_date(planned_start)  # 提取日期部分

    # 判断是工作日还是周末
    is_planned_weekday = is_weekday(planned_date)  # 周一至周五
    is_target_weekday = is_weekday(target_date)

    # 致命检查：工作日/周末必须匹配
    if is_planned_weekday != is_target_weekday:
        # ❌ 致命错误！立即拒绝分配
        raise FatalError(
            f"工单{work_order['工單號']}的日期归属错误！",
            f"计划开始时间：{planned_date}（{'工作日' if is_planned_weekday else '周末'}）",
            f"目标分配日期：{target_date}（{'工作日' if is_target_weekday else '周末'}）",
            "工作日工单绝对禁止分配到周末，周末工单绝对禁止分配到工作日！"
        )

    # ✅ 通过检查，允许分配
    return True
```

### 5. 日期判断规则

**工作日定义：**
- 周一至周五（Mon-Fri）
- 典型时间：18:00-20:00
- 日期示例：05-18, 05-20, 05-21, 05-23

**周末定义：**
- 周六、周日（Sat-Sun）
- 时间任意：可能包含00:30, 01:30等早上时间
- 日期示例：05-24, 05-25

### 6. 生成前强制检查清单

**AI在输出JSON前必须询问自己：**

- [ ] 是否所有工作日工单都分配到了工作日日期列？
- [ ] 是否所有周末工单都分配到了周末日期列？
- [ ] 是否有任何工作日工单错误地出现在周末日期？
- [ ] 是否有任何周末工单错误地出现在工作日日期？
- [ ] 如发现错误，是否立即重新分配？

### 7. 零容忍政策

**工作日/周末分离规则执行零容忍政策：**
- 一次违规 = 整个排班表作废
- 不允许"时间冲突"的借口
- 不允许"人员不足"的理由
- 不允许"看起来合理"的变通
- 必须百分之百遵守日期归属规则

### 8. 常见错误模式（必须避免）

#### ❌ 错误模式1：因为人员不足就跨日期分配
```
错误思维：05-21工单太多，05-24人员空闲，所以把一些工单挪到05-24
正确做法：绝对禁止！宁可05-21一个人承担多个工单，也不能挪到周末
```

#### ❌ 错误模式2：因为时间接近就模糊处理
```
错误思维：05-21晚上和05-24早上时间接近，可以灵活分配
正确做法：绝对禁止！工作日就是工作日，周末就是周末，不能模糊
```

### 9. 记忆强化

**AI必须将以下规则刻入核心逻辑：**

```
IF work_order.計劃開始時間 IS weekday:
    THEN assignment.date MUST BE weekday
    ELSE:
        FATAL_ERROR("工作日工单分配到周末！")

IF work_order.計劃開始時間 IS weekend:
    THEN assignment.date MUST BE weekend
    ELSE:
        FATAL_ERROR("周末工单分配到工作日！")
```
